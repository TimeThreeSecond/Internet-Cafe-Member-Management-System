# 网吧会员管理系统 (Internet Cafe Member Management System) 项目需求文档

## 1. 项目基本信息
- **项目名称**: 网吧会员管理系统
- **项目模式**: 小组项目（6人一组）
- **小组构成**:
    - 组长 (1名): 负责项目框架搭建、任务分派、进度把控、代码管理，并获额外加分。
    - 组员 (5名): 在组长安排下实现具体功能模块。
- **技术栈**: Bootstrap, HTML5, JavaScript, JQuery, CSS, Servlet, JSP, MySQL。
- **项目展示时间**: 暂定第18周。

## 2. 项目考核流程与要求 (总时长不超过15分钟)
| 阶段 | 主讲人 | 核心内容 | 时长要求 |
| :--- | :--- | :--- | :--- |
| 1. 项目文档讲解 | 组长 | 功能模块划分、人员任务安排、完成度、遇到的问题及解决方案。 | 约3分钟 |
| 2. 项目架构介绍 | 组长/组员 | 源码包结构、项目配置细节、如何运行项目。 | 约2分钟 |
| 3. 项目功能演示 | 全体 | 启动并演示项目的各个功能模块。 | 约5分钟 |
| 4. 问题答辩 | 模块负责人 | 回答导师针对特定功能模块的提问。 | 约5分钟 |

## 3. 项目统一技术要求
- **界面**: 使用Bootstrap技术实现自适应的友好界面，能根据不同屏幕尺寸自动调整布局。
- **代码规范**:
    - **包结构**: 逻辑清晰，功能划分合理。
    - **命名规范**: Java类、HTML/JSP页面、JS函数、Java类方法等命名需规范，做到“望文生义”。
- **初始化**: 如需数据库，需提供可执行的初始化SQL文件。

## 4. 功能需求明细

### 4.1 注册与登录模块
- **管理员账号**:
    - 不开放注册，账号密码通过数据库初始化脚本直接设定。
- **普通用户注册**:
    - **必填字段**: 用户名、昵称、密码。
    - **可选字段**: 联系方式（电话/QQ/微信/邮箱）、性别（男/女/保密）。
    - *鼓励小组自行设计扩展其他字段。*

### 4.2 管理员功能模块
- **会员查看**
    - **搜索功能**: 支持按会员等级、电话号码、用户名（模糊匹配）、性别等条件搜索。
    - **列表显示**: 显示账户名、等级、性别、生日、入会日期、总充值金额、已消费金额、账户余额等信息。
    - **详情查看**: 点击账户名可跳转至会员详细信息页面。
- **线下充值**
    - **临时卡充值**: 记录充值金额、支付方式（现金/支付宝/微信等）、充值时间等信息，无需关联会员。
    - **会员卡充值**: 通过查询会员信息，跳转至专属充值页面完成充值。
- **开机/包房**
    - 实现资金划扣，并同步更新用户余额和系统账单。
- **退费管理**
    - **审批列表**: 展示待处理的退费申请。
    - **退费流程**: 支持通过查询会员信息或临时卡信息，进行人工当场退费操作。
- **资金统计**
    - **当日统计**: 按收入方式统计单笔金额和总收入。
    - **当月统计**: 按日统计收入列表、总收入、总退费金额。
    - *鼓励小组对统计字段和形式进行扩展设计。*

### 4.3 会员功能模块
- **个人信息管理**: 可修改账户名、密码、联系方式等信息。
- **消费清单查询**: 可按日期、消费类型（如包时/包房）等条件查询历史消费记录。
- **线上充值**: 通过表单模拟支付流程，充值成功后同步更新用户账户和系统资金账单。
- **退费申请**: 会员可查看余额并在线发起退费申请。

### 4.4 数据库设计
- 数据库表结构及数据模型由小组自行设计实现。
